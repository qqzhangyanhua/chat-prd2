# AI Co-founder V1 设计文档

## 1. 产品定义

AI Co-founder 是一个通过持续对话帮助用户从模糊 idea 收敛到可执行 PRD 的智能体系统。

它的核心价值不是替用户直接写文档，而是像真正的联合创始人一样，持续挑战、澄清、追问和推动决策。

## 2. 产品原则

- 不是普通聊天产品
- 不是死板工作流编排器
- 是持续引导用户挖掘想法的智能体工作台
- 每轮对话都必须推进理解、排除方案或确认决策
- PRD 在对话过程中实时沉淀，而不是最后一次性生成

## 3. 核心体验

用户输入一个初始 idea 后，系统进入持续推进模式：

1. 理解当前输入和上下文
2. 识别信息缺口、风险和含糊点
3. 主动给出判断、挑战或选项
4. 追问一个最关键的问题
5. 更新结构化状态与 PRD 草稿
6. 继续下一轮，直到形成可执行 PRD

重点要求：

- 如果用户描述还不够清晰，智能体必须继续跟进
- 不能把“阶段”做成强制跳转流程
- 阶段只是提示，不是硬编码的漏斗页面

## 4. 系统架构

### 4.1 技术选型

- 前端：Next.js 15、React 19、Tailwind、Zustand
- 后端：FastAPI、SQLAlchemy、Alembic
- 数据层：Postgres
- 流式通信：SSE

### 4.2 逻辑分层

```text
Web Workspace
  -> API Routes
  -> Session / Auth / Export Services
  -> Agent Runtime
  -> State / PRD / Message Repositories
  -> Postgres
```

### 4.3 架构约束

- 不采用自由漫游式多智能体
- 使用单智能体 + 状态驱动推进
- 对话历史不是唯一依据，结构化状态才是主驱动
- 前端围绕“构建产品”而不是“纯聊天”来设计

## 5. 智能体设计

### 5.1 智能体角色

智能体不是执行预设工作流的节点机，而是围绕当前状态持续做三件事：

- 判断用户已经明确了什么
- 识别还缺什么、哪里有风险
- 决定下一轮最应该推动什么

### 5.2 当前运行时能力

当前 V1 已落地的智能体运行时能力包括：

- 基于状态判断下一步动作
- 生成结构化事件流
- 持续更新 PRD 面板
- 保存用户和助手消息

### 5.3 后续增强方向

- 更强的 challenge 策略
- 更明确的状态 patch 生成
- 更细粒度的 PRD section 更新
- 基于历史决策的连续追问

## 6. 状态模型

状态需要满足以下原则：

- 每轮更新
- 可恢复
- 可回溯
- 可用于驱动下一步决策

建议核心字段：

```python
class State(TypedDict):
    idea: str
    goal: str | None
    target_user: str | None
    problem: str | None
    solution: str | None
    assumptions: list[str]
    risks: list[str]
    decisions: list[dict]
    options: list[dict]
    stage_hint: str
    iteration: int
    last_action: str | None
```

这里的 `stage_hint` 只用于帮助界面表达进度感，不代表固定流程。

## 7. PRD 机制

PRD 不是最终一次性生成，而是边对话边沉淀。

建议包含：

- 产品概述
- 目标用户
- 核心问题
- 解决方案
- MVP 范围
- 用户流程
- 成功指标
- 风险
- 待确认问题

当前 V1 已支持：

- PRD sections 的前端实时更新
- 服务端快照读取
- Markdown 导出

## 8. UI 设计

### 8.1 布局

采用三栏工作台：

- 左侧：会话与动作入口
- 中间：结构化对话区
- 右侧：PRD 面板

### 8.2 交互原则

- 选择优先于自由输入
- 每轮必须有推进动作
- 风险和未确认信息要可见
- 用户应感知到自己在“做产品”，而不是“和机器人闲聊”

### 8.3 当前已实现界面能力

- 注册 / 登录页
- 工作台三栏结构
- 流式消息展示
- PRD 面板状态展示
- 会话恢复与导出入口

## 9. MVP 范围

### 必做

- 单用户认证
- 用户绑定会话
- 智能体对话推进
- 结构化状态保存
- PRD 实时更新
- 会话恢复
- Markdown 导出

### 暂不做

- 多人协作
- 外部工具集成
- 复杂权限系统
- 多智能体自治编排
- 完整检索增强系统

## 10. 当前结论

AI Co-founder V1 的正确方向已经明确：

- 产品本质是智能体，不是工作流
- UI 本质是决策工作台，不是聊天框
- 系统本质是状态驱动收敛，而不是依赖长聊天历史
- V1 已经具备从认证、工作台到会话恢复和导出的闭环基础
