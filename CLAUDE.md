## 项目愿景

AI Co-founder 是一个产品工作台：通过持续对话帮助用户挖掘想法、收敛方向，逐步沉淀为可执行的 PRD（产品需求文档）。系统扮演"AI 联合创始人"角色，主动追问、挑战假设、引导决策，最终输出结构化 PRD。

## 架构总览

Monorepo 结构，包含两个核心应用模块：

- **apps/api** -- Python FastAPI 后端，提供认证、会话管理、消息流式返回、PRD 导出、模型配置管理等 REST/SSE API。
- **apps/web** -- Next.js 15 + React 19 前端，提供登录/注册、工作台会话、对话面板、PRD 实时面板、管理后台等 UI。

数据库使用 PostgreSQL（通过 SQLAlchemy ORM + Alembic 迁移）。前后端通过 HTTP JSON API + SSE 通信。后端配备结构化 `ApiError`（含 `recovery_action`），前端通过 `use-schema-gate` hook 和 `recovery-action` 工具层处理错误恢复。

## SSE 事件协议

完整事件序列（发送消息）:

```
message.accepted
  -> reply_group.created
  -> assistant.version.started
  -> action.decided
  -> assistant.delta (多次)
  -> assistant.done
  -> prd.updated (可选)
```

重新生成时复用同一序列，`is_regeneration: true`。

## AI 使用指引

- 修改后端代码时，注意 Repository -> Service -> Route 三层分离，不要跳层
- 修改前端组件时，状态逻辑在 store 中处理，组件只负责渲染和事件分发
- Agent 模块 (`apps/api/app/agent/`) 已具备规则引擎 + LLM 模型网关双路径（`reply_mode: "gateway" | "local"`）
- 错误系统：后端抛出 `ApiError`（含 `code`, `message`, `recovery_action`），前端通过 `lib/recovery-action.ts` 解析并在 `WorkspaceErrorNotice` 中渲染可点击动作
- Schema 检测：前端 `useSchemaGate` hook 在进入工作台前检查 `/api/health`，若 schema 为 `outdated` 则展示 `SchemaOutdatedNotice`，阻止操作直到迁移完成
- 管理端：`/api/admin/model-configs` 需 Bearer token 且邮箱在 `ADMIN_EMAILS` 中
- 测试：后端用 SQLite in-memory，前端用 jsdom 模拟浏览器；PBT 用 fast-check 测试分组逻辑边界
